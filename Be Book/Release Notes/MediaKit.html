get_next_encoder()

                                                         
  

status_t get_next_encoder(int32 *cookie, media_codec_info *ei);

status_t get_next_encoder( int32 *cookie, const media_file_format *mfi, const media_format *input_format, const media_format *output_format, media_codec_info *ei, media_format *accepted_input_format, media_format *accepted_output_format)

  • mfi comes from get_next_file_format(), pass NULL if you don't care.

  • input_format is the type of data given to the encoder, wildcards are allowed.

  • output_format: the desired output format, wildcards allowed.

  • ei: information about the encoder.

  • accepted_input_format: Type of data that the encoder will accept as input. Wildcards given in input_format will be specialized here.

  • accepted_output_format: The type of data that the encoder will output. Wildcards in output_format will be specialized here.

Constants and Types

Declared in: media/MediaDefs.h

  • new B_MEDIA_FLAVORS_CHANGED value for use as >what> field of messages.

  • new media_multi_channels and media_multi_matrix enumerations for use with multiple audio channels.

  • changes to media_raw_audio_format enumeration.

  • media_raw_audio_format is deprecated. Please use media_multi_audio_format instead. media_format.u.raw_audio and media_raw_audio_format::wildcard have changed to a media_multi_audio_format struct, which is upwards compatible.

  • new media_multi_audio_info struct (used for a new field in media_encoded_audio_format)

  • new media_display_flags struct

  • new media_format_flags enumeration

  • changes to the media_format struct, including these new inline functions: Width(), Height(), and Encoding().

    plus changes in constness of existing functions.

  • new B_AVR_FORMAT_FAMILY in the media_format_family typedef

  • new structs for use by encoders and file writers: media_encode_info, encode_parameters, media_decode_info
